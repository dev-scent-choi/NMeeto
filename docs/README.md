# NMeeto

AI 면접관과 실제 면접처럼 대화하며 연습하는 모의면접 서비스.

회사·직무·이력서를 입력하면 AI가 맞춤 질문을 생성하고, 답변 내용에 따라 꼬리질문을 던지며
실제 면접과 유사한 대화를 진행한다. 종료 후에는 답변 구조·발화 습관·직무 적합도를 분석한
피드백 리포트를 제공한다.

## 현재 상태

**설계 단계.** 코드는 아직 없으며, `docs/`의 기획·명세 문서가 개발 착수 기준이다.

## 핵심 설계 원칙

1. **흐름은 코드가, 내용은 LLM이.** 면접 진행 전체를 LLM에 맡기면 시간 관리와 질문 커버리지가
   무너진다. 상태머신이 단계를 통제하고 LLM은 발화 생성만 담당한다.
2. **역할별 모델 분리.** Question Planner / Interviewer / Turn Judge / Evaluator를 각각 다른
   등급의 모델로 호출한다. 하나의 프롬프트로 전부 처리하면 품질과 비용이 동시에 나빠진다.
3. **프롬프트는 코드가 아니라 자산.** `packages/prompts/`에 파일로 분리해 버전 관리하고,
   세션마다 사용된 버전을 기록한다.
4. **Eval 없이는 개선도 없다.** 프롬프트 변경의 품질 영향을 측정할 수 없으면 이 제품은
   개선이 불가능하다. Eval 하네스는 축소 대상이 아니다.

## 개발 순서

- **Phase 0** — 기술 검증: 한국어 STT 벤치마크(스트리밍 partial 지원 포함), TTS 보이스 선정,
  꼬리질문 프롬프트 프로토타입. *꼬리질문 평가 3.5/5 미만이면 Phase 1 착수 금지.*
- **Phase 1** — 텍스트 면접 MVP: 이력서 파싱, 질문 계획, 대화형 진행, 피드백 리포트
- **Phase 2** — 음성: 스트리밍 STT/TTS, VAD, 패널 면접, 압박 모드
- **Phase 3** — 실시간·화상: LiveKit 전환, barge-in, 지원자 녹화 분석

상세 완료 조건은 구현명세서 §19 Phase별 DoD 참조.

## 착수 전 미결정 사항

구현명세서 §21 참조. 특히 **영어 면접 지원 여부**는 프롬프트·질문 풀 구조에 영향을 주므로
Phase 1 착수 전에 결정이 필요하다.
